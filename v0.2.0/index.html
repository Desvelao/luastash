Module luastash

LuaStash Module This module provides functionality to process data through a pipeline of generators, filters,and outputs.

It includes logging and error handling mechanisms.

Class Logger

Logger class for logging messages with different levels (debug, info, warn, error). It supports enabling/disabling logging and creating child loggers with specific names and levels.
Logger:new (options)
Create a new Logger instance.

Parameters:

  • options A table of options to configure the logger.
    • name The name of the logger. (default '')
    • level The logging level (debug, info, warn, error). (default 'info')
    • enabled Whether the logger is enabled. (default true)

Returns:

    Logger A new Logger instance configured with the provided options.
Logger:set_level (level)
Set the logging level for the logger instance. The level can be one of "debug", "info", "warn", or "error". If an invalid level is provided, an error will be raised.

Parameters:

  • level The logging level to set (debug, info, warn, error).
Logger:disable ()
Disable logging for this logger instance. When disabled, no log messages will be printed regardless of the logging level.

Returns:

    nil
Logger:enable ()
Enable logging for this logger instance. When enabled, log messages will be printed according to the configured logging level.

Returns:

    nil
Logger:get_logger (options)
Create a child logger with specific options. The child logger will inherit the logging level and enabled state from the parent logger unless overridden in the options. The name of the child logger will be a combination of the parent logger's name and the provided name in the options.

Parameters:

  • options A table of options to configure the child logger.
    • name The name of the child logger, which will be combined with the parent logger's name. (default '')
    • level The logging level for the child logger. If not provided, it will inherit the parent's logging level. (default '')
    • enabled Whether the child logger is enabled. If not provided, it will inherit the parent's enabled state. (default true)

Returns:

    Logger A new Logger instance configured with the provided options and inheriting from the parent logger.

Class Event

Event class representing a data event in the pipeline. It provides methods to get and set fields using Logstash-style paths, manage metadata, clone events, and serialize to JSON.
Event:new ([data=nil])
Create a new Event instance.

Parameters:

  • data data Initial event data, defaults to an empty table if not provided. (default nil)

Returns:

    Event A new Event instance with the provided data and default metadata.

Usage:

    local event = Event:new({foo = "bar"})
Event:get (path)
Get field using "[foo][bar]" syntax. If the path does not exist, it returns nil.

Parameters:

  • path string The field path in Logstash-style syntax (e.g., "[foo][bar]").

Returns:

    The value at the specified path, or nil if the path does not exist.

Usage:

    local value = event:get("[foo][bar]")
Event:set (path, value)
Set field using "[foo][bar]" syntax. If intermediate tables do not exist, they will be created.

Parameters:

  • path string The field path in Logstash-style syntax (e.g., "[foo][bar]").
  • value The value to set at the specified path. If intermediate tables do not exist, they will be created.

Usage:

    event:set("[foo][bar]", "value")

event.metadata
Event metadata fields. These fields provide additional information about the event and are used for processing and routing within the stash pipeline. The metadata includes standard Logstash fields such as @timestamp and @version, as well as custom fields like source and source_tag that indicate the origin of the event.

Fields:

  • timestamp The timestamp of the event, defaulting to the current time in ISO 8601 format if not provided in the initial data.
  • version The version of the event, defaulting to "1" if not provided in the initial data.
  • source The input type of the event.
  • source_tag The tag of the input if provided in the configuration.
event.data
Event data fields. These fields contain the actual data of the event that is processed through the stash pipeline. The data can be structured in a nested manner, and fields can be accessed and modified using Logstash-style paths (e.g., "[foo][bar]").
  • data The main data table of the event, which can contain any structured data relevant to the event being processed.

Run Stash

StashConfig
Pipeline configuration table.

Fields:

  • inputs List of input processor configurations. Each configuration should include a type field that corresponds to a processor function in the inputs table of the StashProcessors.
  • filters List of filter processor configurations. Each configuration should include a type field that corresponds to a processor function in the filters table of the StashProcessors.
  • outputs List of output processor configurations. Each configuration should include a type field that corresponds to a processor function in the outputs table of the StashProcessors.

Usage:

    local config = {
    inputs = {
        { type = "input_type_1", options = { ... } },
        { type = "input_type_2", options = { ... } },
    },
    filters = {
        { type = "filter_type_1", options = { ... } },
        { type = "filter_type_2", options = { ... } },
    },
    outputs = {
        { type = "output_type_1", options = { ... } },
        { type = "output_type_2", options = { ... } },
    },
    options = { ... },
}
StashProcessors
Define the inputs, filters and outputs processors. This table should contain functions for each processor type used in the stash pipeline.

Fields:

  • inputs Defines input processor functions.
  • filters Defines filter processor functions.
  • outputs Defines output processor functions.

call (config, processors[, context=nil[, utils=nil]])
Runs the stash pipeline. This function processes data through a series of generators, filters and outputs.

Parameters:

  • config StashPipelineConfig or string Pipeline configuration table or file path
  • processors StashProcessors Inputs, filters and outputs processors
  • context any Context object passed to processors (default nil)
  • utils Utility functions and logger
    • logger Logger instance for logging within processors (default nil)

Returns:

    nil

Usage:

    local luastash = require("luastash")
